# Windows-11-adversary-simulation-lab

Project Overview 
This project is documenting the creation of an isolated cybersecurity lab containging a Kali Linux testing Machine and a Windows 11 target amchine. 

The Purpose of the lab is to practice authorized adversary simulation, observe how security events appear on a windows endpoint, and study how defensive tools can detect and investigate suspcisous activity.

All testing is performed only inside virtual machines that I won and control.  The lab is compelelty isolated from production devices external systems and public networks.

Project Overview
- Build a safe and Isolated virtual Cybersecurity envrionment.
- Configure Kali Liux as the security-testing machine (The Attacker).
- Configure Windows 11 as the monitored endpoint (The Victim).
- Practice service network discovery and service enmumeration.
- Simulate controlled command and control behavior.
- Study reverse-shell activity in an authorized lab.
- Examine keystroke logging behavior for a defensive detection persepective.
- Collect screenshots, logs, and indicators of compromise.
- Document detection, investigation, and remediations steps.
- Improve practical networking, Linux, Windows, and incident repsose skills.
